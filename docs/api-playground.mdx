## Overview

<Warning>
  **Security**: Never use production credentials in public documentation. Use UAT credentials only.
</Warning>

## Setup Your Credentials

### Step 1: Encode Your Credentials

Your API credentials must be Base64 encoded for Basic Authentication.

<CodeGroup>

```bash Terminal
echo -n "your-username:your-password" | base64
```

```javascript JavaScript
const credentials = btoa('your-username:your-password');
console.log(credentials);
```

```python Python
import base64
credentials = base64.b64encode(b'your-username:your-password').decode('utf-8')
print(credentials)
```

</CodeGroup>

## Postman Collection

### Import Collection

Download and import our Postman collection for easy testing:

<Steps>
  <Step title="Download Collection">
    Download the [CT EVV API Postman Collection](./postman/ct-evv-api.json)
  </Step>
  
  <Step title="Import to Postman">
    1. Open Postman
    2. Click "Import"
    3. Select the downloaded JSON file
    4. Collection will be added to your workspace
  </Step>
  
  <Step title="Configure Environment">
    1. Create new environment in Postman
    2. Add variables:
       - `base_url`: `https://uat-api.sandata.com/interfaces/intake`
       - `auth_token`: Your Base64 encoded credentials
       - `account_id`: Your account ID
       - `provider_id`: Your 9-digit provider ID
  </Step>
  
  <Step title="Test Endpoints">
    1. Select environment
    2. Choose endpoint from collection
    3. Click "Send"
    4. Review response
  </Step>
</Steps>

## Automated Testing Script

### Node.js Test Script

```javascript
const axios = require('axios');

const config = {
  baseURL: 'https://uat-api.sandata.com/interfaces/intake',
  headers: {
    'Authorization': 'Basic YOUR_ENCODED_CREDENTIALS',
    'Account': 'YOUR_ACCOUNT_ID',
    'Content-Type': 'application/json'
  }
};

async function testAPI() {
  try {
    // 1. Create Employee
    console.log('Creating employee...');
    const empResponse = await axios.post(
      '/employees/rest/api/v1.1',
      [{ /* employee data */ }],
      config
    );
    const empUUID = empResponse.data.data.uuid;
    console.log(`Employee UUID: ${empUUID}`);
    
    // 2. Wait and check status
    await new Promise(resolve => setTimeout(resolve, 5 * 60 * 1000));
    const empStatus = await axios.get(
      `/employees/rest/api/v1.1/status?uuid=${empUUID}`,
      config
    );
    console.log(`Employee Status: ${empStatus.data.messageSummary}`);
    
    // 3. Create Client
    console.log('Creating client...');
    const clientResponse = await axios.post(
      '/clients/rest/api/v1.1',
      [{ /* client data */ }],
      config
    );
    const clientUUID = clientResponse.data.data.uuid;
    console.log(`Client UUID: ${clientUUID}`);
    
    // Continue with visit...
    
  } catch (error) {
    console.error('Error:', error.response?.data || error.message);
  }
}

testAPI();
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="401 Unauthorized">
    **Problem**: Authentication failed
    
    **Solutions**:
    - Verify credentials are correct
    - Check Base64 encoding
    - Ensure Authorization header format is correct
    - Verify Account ID matches credentials
  </Accordion>
  
  <Accordion title="403 Forbidden">
    **Problem**: Account mismatch
    
    **Solutions**:
    - Verify Account ID is correct
    - Check EntityGuid if multi-account
    - Ensure credentials match account
    - Contact support if persists
  </Accordion>
  
  <Accordion title="422 Validation Error">
    **Problem**: Data validation failed
    
    **Solutions**:
    - Review error message in response
    - Check required fields are present
    - Verify data formats are correct
    - Test with minimal valid payload first
  </Accordion>
  
  <Accordion title="UUID Not Found">
    **Problem**: Status check returns not found
    
    **Solutions**:
    - Verify UUID is correct
    - Check UUID hasn't expired (48 hours)
    - Ensure POST was successful
    - Try resubmitting if needed
  </Accordion>
</AccordionGroup>

## Best Practices

<AccordionGroup>
  <Accordion title="Use UAT for Testing">
    - Never test in production
    - Use UAT environment for all development
    - Test thoroughly before moving to production
    - Keep UAT and production credentials separate
  </Accordion>
  
  <Accordion title="Unique Test IDs">
    - Use timestamps or UUIDs for test data
    - Avoid reusing test IDs
    - Clean up test data periodically
    - Document test data patterns
  </Accordion>
  
  <Accordion title="Automated Testing">
    - Create automated test suites
    - Test all endpoints regularly
    - Validate responses programmatically
    - Monitor for API changes
  </Accordion>
  
  <Accordion title="Error Handling">
    - Log all requests and responses
    - Handle all error scenarios
    - Implement retry logic
    - Alert on failures
  </Accordion>
</AccordionGroup>
