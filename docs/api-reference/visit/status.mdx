## Overview

After submitting visit data via POST, use this endpoint to check if the submission has been processed successfully or if there were any errors.

<Info>
  **Wait Time**: Wait at least 5 minutes after POST before checking status. The UUID is available for 48 hours.
</Info>

## Endpoint

<CodeGroup>

```bash Production
GET https://api.sandata.com/interfaces/intake/visits/rest/api/v1.1/status?uuid={uuid}
```

```bash UAT
GET https://uat-api.sandata.com/interfaces/intake/visits/rest/api/v1.1/status?uuid={uuid}
```

</CodeGroup>

## Common Visit Error Codes

| Error Code | Meaning | Solution |
|------------|---------|----------|
| `-100` | Client not found | Create client first |
| `-101` | Employee not found | Create employee first |
| `TIME-01` | CallOut ≤ CallIn | Fix call times |
| `TIME-02` | AdjOut ≤ AdjIn | Fix adjusted times |
| `TIME-03` | Invalid time combination | Review call logic rules |
| `-709` | Duplicate SequenceID | Increment SequenceID |
| `TASK-01` | Missing required tasks | Add tasks for program |
| `CHANGE-01` | Missing VisitChanges | Add VisitChanges segment |
