## Overview

The Employee endpoint receives information about caregivers who provide services. Employees must be successfully created before submitting visits.

<Info>
  **Important**: This endpoint handles both new employee creation and updates to existing employees. Use incrementing `SequenceID` values for updates.
</Info>

## Endpoint

<CodeGroup>

```bash Production
POST https://api.sandata.com/interfaces/intake/employees/rest/api/v1.1
```

```bash UAT
POST https://uat-api.sandata.com/interfaces/intake/employees/rest/api/v1.1
```

</CodeGroup>

## Authentication

<ParamField header="Authorization" type="string" required>
  Basic Authentication header
  
  Format: `Basic <base64(username:password)>`
</ParamField>

<ParamField header="Account" type="string" required>
  Your Sandata account ID (or use `EntityGuid` for multi-account)
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`
</ParamField>

## Request Body

The request body must be a JSON array of employee objects. Each transaction can contain 1-5,000 employees.

### Provider Identification (Required)

<ParamField body="ProviderIdentification" type="object" required>
  <Expandable title="properties">
    <ParamField body="ProviderQualifier" type="string" required>
      Type of provider identifier
      
      **Value**: `MedicaidID`
    </ParamField>
    
    <ParamField body="ProviderID" type="string" required>
      9-digit DSS-AVRS Medicaid ID
      
      **Format**: `#########` (9 digits)
      
      **Max Length**: 64 characters
    </ParamField>
  </Expandable>
</ParamField>

### Employee General Information (Required)

<ParamField body="EmployeeQualifier" type="string" required>
  Type of employee identifier being sent
  
  **Value**: `EmployeeSSN`
  
  **Note**: Must be mixed case exactly as shown
  
  **Max Length**: 20 characters
</ParamField>

<ParamField body="EmployeeIdentifier" type="string" required>
  Employee identifier in SSN-based format
  
  **Format**: `#########` (4 generic leading digits + last 5 digits of SSN)
  
  **Example**: `000012345`
  
  **Max Length**: 9 characters
  
  **Note**: Must match `EmployeeSSN` value
</ParamField>

<ParamField body="EmployeeOtherID" type="string">
  Unique employee identifier from your external system
  
  **Max Length**: 64 characters
  
  **Note**: Use your system's internal employee ID for easier tracking
</ParamField>

<ParamField body="SequenceID" type="integer" required>
  Sequence number for this employee record
  
  **Rules**:
  - First submission: Use 1
  - Each update: Increment by 1 or more
  - Can use timestamp format: `YYYYMMDDHHMMSS`
  - Must be unique per employee
  - Gaps are allowed
  
  **Max Length**: 16 digits
  
  **Example**: `1`, `2`, `3` or `20240115093045`
</ParamField>

<ParamField body="EmployeeSSN" type="string" required>
  Employee's SSN in required format
  
  **Format**: `#########` (4 generic leading digits + last 5 digits of SSN)
  
  **Example**: `000012345`
  
  **Max Length**: 9 characters
  
  **Note**: Must match `EmployeeIdentifier` value
</ParamField>

<ParamField body="EmployeeLastName" type="string" required>
  Employee's last name
  
  **Max Length**: 30 characters
  
  **Allowed Characters**:
  - Alpha letters
  - Hyphens (-)
  - Periods (.)
  - Apostrophes (')
  
  All other special characters will be rejected
</ParamField>

<ParamField body="EmployeeFirstName" type="string" required>
  Employee's first name
  
  **Max Length**: 30 characters
  
  **Allowed Characters**:
  - Alpha letters
  - Hyphens (-)
  - Periods (.)
  - Apostrophes (')
  
  All other special characters will be rejected
</ParamField>

<ParamField body="EmployeeEmail" type="string">
  Employee's email address
  
  **Max Length**: 64 characters
  
  **Conditional**: Required if available
  
  **Format Requirements**:
  - Must contain "@" symbol
  - Must have valid extension (.com, .org, etc.)
  
  **Example**: `john.smith@example.com`
</ParamField>

## Business Rules

<Warning>
  **Critical Validation Rules**
</Warning>

### Required Fields

The following fields are **always required**:
- `ProviderIdentification` (complete object)
- `EmployeeQualifier`
- `EmployeeIdentifier`
- `SequenceID`
- `EmployeeSSN`
- `EmployeeLastName`
- `EmployeeFirstName`

### SSN Format Rules

- **EmployeeSSN** and **EmployeeIdentifier** must match exactly
- Must be exactly 9 digits
- Format: 4 generic leading digits + 5 SSN digits
- Example: `000012345`
- No dashes, spaces, or other characters

### Name Validation

- **EmployeeFirstName** and **EmployeeLastName** only allow:
  - Letters (A-Z, a-z)
  - Hyphens (-)
  - Periods (.)
  - Apostrophes (')
- No numbers or other special characters
- Max 30 characters each

### Email Validation

- If provided, must be valid format
- Must contain "@" symbol
- Must have extension (.com, .org, etc.)
- Max 64 characters
- Invalid email will cause rejection

### Sequence Rules

- First submission for an employee: SequenceID = 1
- Each update: Increment SequenceID
- Duplicate SequenceID will be rejected
- Lower SequenceID than previously processed will be accepted but logged as historical

## Common Errors

| Error | Cause | Solution |
|-------|-------|----------|
| Missing EmployeeSSN | EmployeeSSN not provided | Add 9-digit SSN format |
| Invalid Name Characters | Numbers or special chars in name | Remove invalid characters |
| SSN Mismatch | EmployeeIdentifier â‰  EmployeeSSN | Ensure both fields match |
| Duplicate Sequence | SequenceID already used | Increment SequenceID |
| Invalid Email | Email missing @ or extension | Fix email format |
| Missing First/Last Name | Name fields not provided | Add required name fields |
