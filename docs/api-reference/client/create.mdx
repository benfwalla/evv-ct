## Overview

The Client endpoint receives information about individuals receiving care. Clients must be successfully created before submitting visits.

<Info>
  **Important**: This endpoint handles both new client creation and updates to existing clients. Use incrementing `SequenceID` values for updates.
</Info>

## Endpoint

<CodeGroup>

```bash Production
POST https://api.sandata.com/interfaces/intake/clients/rest/api/v1.1
```

```bash UAT
POST https://uat-api.sandata.com/interfaces/intake/clients/rest/api/v1.1
```

</CodeGroup>

## Request Body

The request body must be a JSON array of client objects. Each transaction can contain 1-5,000 clients.

### Provider Identification (Required)

<ParamField body="ProviderIdentification" type="object" required>
  <Expandable title="properties">
    <ParamField body="ProviderQualifier" type="string" required>
      Type of provider identifier
      
      **Value**: `MedicaidID`
    </ParamField>
    
    <ParamField body="ProviderID" type="string" required>
      9-digit DSS-AVRS Medicaid ID
      
      **Format**: `#########` (9 digits)
      
      **Max Length**: 64 characters
    </ParamField>
  </Expandable>
</ParamField>

### Client General Information (Required)

<ParamField body="ClientQualifier" type="string" required>
  Type of client identifier being sent
  
  **Value**: `ClientCustomID`
  
  **Note**: Must be mixed case exactly as shown
</ParamField>

<ParamField body="ClientIdentifier" type="string" required>
  Unique 9-digit Medicaid ID for the client
  
  **Format**: `#########`
  
  **Max Length**: 64 characters
  
  **Note**: Must match `ClientCustomID` value
</ParamField>

<ParamField body="ClientFirstName" type="string" required>
  Client's first name
  
  **Max Length**: 30 characters
  
  **Allowed Characters**:
  - Alpha letters
  - Hyphens (-)
  - Periods (.)
  - Apostrophes (')
  
  All other special characters will be rejected
</ParamField>

<ParamField body="ClientMiddleInitial" type="string">
  Client's middle initial
  
  **Max Length**: 1 character
</ParamField>

<ParamField body="ClientLastName" type="string" required>
  Client's last name
  
  **Max Length**: 30 characters
  
  **Allowed Characters**:
  - Alpha letters
  - Hyphens (-)
  - Periods (.)
  - Apostrophes (')
  
  All other special characters will be rejected
</ParamField>

<ParamField body="ClientMedicaidID" type="string" required>
  9-digit Medicaid ID provided by the state
  
  **Format**: `#########`
  
  **Max Length**: 64 characters
</ParamField>

<ParamField body="MissingMedicaidID" type="string">
  Indicator for newborn patients without Medicaid ID yet
  
  **Value**: `"false"` (default)
  
  **Max Length**: 5 characters
</ParamField>

<ParamField body="SequenceID" type="integer" required>
  Sequence number for this client record
  
  **Rules**:
  - First submission: Use 1
  - Each update: Increment by 1 or more
  - Can use timestamp format: `YYYYMMDDHHMMSS`
  - Must be unique per client
  - Gaps are allowed
  
  **Max Length**: 16 digits
  
  **Example**: `1`, `2`, `3` or `20240115093045`
</ParamField>

<ParamField body="ClientOtherID" type="string">
  Additional user-defined client ID from your system
  
  **Max Length**: 24 characters
  
  **Note**: No special characters recommended
</ParamField>

<ParamField body="ClientTimeZone" type="string" required>
  Client's primary time zone
  
  **Value**: `"US/Eastern"` (for Connecticut)
  
  **Max Length**: 64 characters
</ParamField>

<ParamField body="ClientCustomID" type="string" required>
  Same as ClientIdentifier - 9-digit Medicaid ID
  
  **Format**: `#########`
  
  **Max Length**: 24 characters
</ParamField>

<ParamField body="ClientSSN" type="integer">
  Last 5 digits of client's Social Security Number
  
  **Format**: `0000#####` (4 leading zeros + 5 digits)
  
  **Max Length**: 9 digits
  
  **Note**: Not required if ClientOtherID is provided
</ParamField>

      - Spaces

    <ParamField body="ClientAddressLine2" type="string">

    <ParamField body="ClientCounty" type="string">

      **Max Length**: 25 characters
      
      **Allowed Characters**:

      - Hyphens (-)
      - Periods (.)
      - Apostrophes (')
      - Underscores (_)
      - Spaces
    </ParamField>
    
    <ParamField body="ClientCity" type="string" required>
      City name
      
      **Max Length**: 30 characters

## Business Rules

<Warning>
  **Critical Validation Rules**
</Warning>

### Required Fields

The following fields are **always required**:
- `ProviderIdentification` (complete object)
- `ClientQualifier`
- `ClientIdentifier`
- `ClientFirstName`
- `ClientLastName`
- `ClientMedicaidID`
- `SequenceID`
- `ClientTimeZone`
- `ClientCustomID`
- At least one complete `ClientAddress`

### Address Requirements

- At least one address with all required fields (Line1, City, State, Zip)
- Exactly one address must have `ClientAddressIsPrimary` = `"true"`
- Multiple addresses of the same type are allowed

### Sequence Rules

- First submission for a client: SequenceID = 1
- Each update: Increment SequenceID
- Duplicate SequenceID will be rejected
- Lower SequenceID than previously processed will be accepted but logged as historical

### Field Validation

- **ClientIdentifier** must equal **ClientCustomID**
- **ClientMedicaidID** must be 9 digits
- **ClientFirstName** and **ClientLastName** only allow: letters, hyphens, periods, apostrophes
- **ClientState** must be uppercase 2-letter code
- **ClientZip** must be exactly 9 digits

After submitting clients:

1. **Check Status**: Use the UUID to check processing status
2. **Create Employees**: Submit employee records
3. **Submit Visits**: Create visits linking clients and employees
