## Setup Your Environment

Before you begin, ensure you have:

<AccordionGroup>
  <Accordion icon="key" title="API Credentials">
    Contact your Sandata implementation manager to receive:
    - Username and password for Basic Authentication
    - Account ID or Entity GUID
    - Provider ID (DSS-AVRS Medicaid ID)
  </Accordion>

  <Accordion icon="globe" title="VPN Access">
    **Important**: All API requests must originate from within the USA. International requests will be rejected.
    
    Ensure your systems are configured to send requests from US-based servers or through a US VPN.
  </Accordion>

  <Accordion icon="server" title="Environment URLs">
    **UAT (Testing)**:
    - Clients: `https://uat-api.sandata.com/interfaces/intake/clients/rest/api/v1.1`
    - Employees: `https://uat-api.sandata.com/interfaces/intake/employees/rest/api/v1.1`
    - Visits: `https://uat-api.sandata.com/interfaces/intake/visits/rest/api/v1.1`

    **Production**:
    - Clients: `https://api.sandata.com/interfaces/intake/clients/rest/api/v1.1`
    - Employees: `https://api.sandata.com/interfaces/intake/employees/rest/api/v1.1`
    - Visits: `https://api.sandata.com/interfaces/intake/visits/rest/api/v1.1`
  </Accordion>
</AccordionGroup>

## Make Your First Request

<Steps>
  <Step title="Create an Employee">
    Employees must be created before they can be associated with visits.

    ```bash
    curl -X POST https://uat-api.sandata.com/interfaces/intake/employees/rest/api/v1.1 \
      -H "Authorization: Basic <base64_encoded_credentials>" \
      -H "Account: <your_account_id>" \
      -H "Content-Type: application/json" \
      -d '[{
        "ProviderIdentification": {
          "ProviderQualifier": "MedicaidID",
          "ProviderID": "123456789"
        },
        "EmployeeQualifier": "EmployeeSSN",
        "EmployeeIdentifier": "000012345",
        "EmployeeOtherID": "EMP001",
        "SequenceID": 1,
        "EmployeeSSN": "000012345",
        "EmployeeLastName": "Smith",
        "EmployeeFirstName": "John",
        "EmployeeEmail": "john.smith@example.com"
      }]'
    ```

    <ResponseField name="Response" type="object">
      ```json
      {
        "id": "7f6dcd1a-ec5e-4efd-a2d4-1049756016a5",
        "status": "SUCCESS",
        "messageSummary": "The result for the input UUID is not ready yet. Please try again.",
      ```

  <Step title="Check Processing Status">
    Use the UUID from the response to check if processing completed.

    ```bash
    curl -X GET "https://uat-api.sandata.com/interfaces/intake/employees/rest/api/v1.1/status?uuid=7f6dcd1a-ec5e-4efd-a2d4-1049756016a5" \
      -H "Authorization: Basic <base64_encoded_credentials>" \
      -H "Account: <your_account_id>"
    ```

    <Info>
      Wait at least 5 minutes before checking status. The UUID is available for 48 hours.
    </Info>

## Best Practices

<Warning>
  **Important Considerations**:
  - Always send employees and clients before visits
  - Use incremental sequence IDs for each record update
  - Submit visits in near real-time (within 24 hours)
  - Wait 5 minutes between POST and status checks
  - Keep transaction sizes between 1-5,000 records
</Warning>
